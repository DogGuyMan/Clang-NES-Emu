#### 주소 공간(Address Space)
* CPU가 접근할 수 있는 모든 주소의 범위
* 주소 버스의 비트 수로 결정된다
  * 16-bit 주소 버스 → 2^16 = 65536개의 주소 (NES CPU)
  * 32-bit 주소 버스 → 2^32 = 약 4GB
* 메모리의 저장 단위는 **바이트** — 각 주소에 1바이트(u8)가 대응

#### 물리적 주소(Physical Address)
* 실제 메모리 하드웨어의 인덱스
* 물리적으로 존재하는 메모리 칩의 실제 위치

#### 논리적 주소(Logical Address)
* CPU가 생성하는 주소 (가상 주소)
* 물리적 주소와 반드시 1:1 대응하지 않음

#### 주소 바인딩(Address Binding)
* 논리적 주소 → 물리적 주소로 변환하는 과정
* 컴파일 타임, 로드 타임, 실행 타임에 일어날 수 있음

### 오답 노트: NES Bus의 `ram[2048]`을 이해하면서 헷갈렸던 것

#### 혼동했던 것
"주소 공간의 크기"와 "물리 메모리의 크기"를 같은 것으로 착각했다.

* 잘못된 생각: CPU 주소 공간이 0x0000~0xFFFF(64KB)이니까, `ram` 배열도 64KB여야 한다
* 잘못된 생각: `u8 ram[256]`이면 `2^8 * 2^8 = 65536`이니 주소 를 다 표현할 수 있으니 충분하다

#### 핵심 구분

| 개념           | 의미                               | NES에서                 |
| -------------- | ---------------------------------- | ----------------------- |
| 주소 공간      | CPU가 **가리킬 수 있는 위치의 수** | 2^16 = 65536개          |
| 각 위치의 크기 | 한 주소에 저장되는 **값의 크기**   | 1바이트 (u8, 0x00~0xFF) |
| 물리 RAM       | 실제 존재하는 **메모리 칩의 용량** | 2KB (2048바이트)        |

#### NES에 대입하면

* `ram[2048]`은 주소 공간 전체를 표현하는 것이 아니라, **$0000-$07FF 구간만 담당하는 물리 RAM 칩**을 에뮬레이션한 것
* `bus_cpu_read(addr)`은 주소를 보고 **어느 하드웨어에 전달할지 라우팅**하는 역할 — 이것이 주소 디코딩

#### 비유
주소 공간은 "우편번호 체계" (65536개의 우편번호가 존재)
물리 RAM은 "우편번호 0000~07FF에 있는 하나의 아파트 단지"
